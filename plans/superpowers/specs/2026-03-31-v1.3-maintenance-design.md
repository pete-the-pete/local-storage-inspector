# v1.3.0 — Maintenance & Hardening

## Overview

Maintenance release addressing bugs, CI gaps, security hardening, and code quality improvements found during a full codebase audit.

## 1. Bug Fixes

### 1a. Fix keydown listener memory leak

**File:** `src/sidepanel/components/App.tsx:201-203`

**Problem:** `window.addEventListener("keydown", handleKeyDown)` is called directly in the component body — runs on every render, stacking duplicate listeners without cleanup.

**Fix:** Move into a `useEffect` with cleanup:

```tsx
useEffect(() => {
  window.addEventListener("keydown", handleKeyDown);
  return () => window.removeEventListener("keydown", handleKeyDown);
}, [handleKeyDown]);
```

Remove the bare `if (typeof window !== "undefined")` block.

### 1b. Fix ValueEditor stale closure

**File:** `src/sidepanel/components/ValueEditor.tsx:28-72`

**Problem:** `initEditor` callback captures `handleSave` but dependency array is `[jsonMode]`. The Mod-s keymap fires a stale version of `handleSave`.

**Fix:** Use a ref for the save handler so the keymap always calls the current version:

```tsx
const handleSaveRef = useRef(handleSave);
handleSaveRef.current = handleSave;
```

Then in the keymap: `handleSaveRef.current()`. This avoids recreating the entire CodeMirror editor when the save handler changes.

## 3. Security

### 3a. Validate messages in monitor.ts

**File:** `src/content/monitor.ts:36-50`

**Problem:** `window.postMessage` events with `_lsi: "interceptor"` are trusted without validating field types or enum values. A malicious page could inject fake change events.

**Fix:** Add a validation function in `src/lib/validate.ts` (extends existing file):

```typescript
export function isValidStorageChangeData(data: unknown): boolean {
  if (typeof data !== "object" || data === null) return false;
  const d = data as Record<string, unknown>;
  const validStorageTypes = ["localStorage", "sessionStorage"];
  const validOperations = ["setItem", "removeItem", "clear"];
  const validSources = ["page", "extension", "unknown"];
  return (
    validStorageTypes.includes(d.storageType as string) &&
    validOperations.includes(d.operation as string) &&
    validSources.includes(d.source as string) &&
    typeof d.timestamp === "number" &&
    (typeof d.key === "string" || d.key === null) &&
    (typeof d.oldValue === "string" || d.oldValue === null) &&
    (typeof d.newValue === "string" || d.newValue === null)
  );
}
```

Use in monitor.ts:

```typescript
if (event.data?._lsi !== "interceptor") return;
if (!isValidStorageChangeData(event.data)) return;
```

### 3b. Validate sender in App.tsx

**File:** `src/sidepanel/components/App.tsx:87-95`

**Problem:** `chrome.runtime.onMessage` doesn't verify the sender is our own content script.

**Fix:** Check `sender.id`:

```typescript
const handleMessage = (
  message: StorageChangePortMessage,
  sender: chrome.runtime.MessageSender,
) => {
  if (sender.id !== chrome.runtime.id) return;
  if (message.type === "STORAGE_CHANGE" && recordingRef.current) {
    addChanges(message.changes);
  }
};
```

### 4b. Extract Chrome API helpers to src/lib/storage.ts

**File:** `src/sidepanel/components/App.tsx:15-55`

**Problem:** 5 functions that are Chrome API helpers live inside the component file, violating "pure functions in src/lib/" convention.

**Fix:** Create `src/lib/storage.ts` with:
- `getActiveTabId(): Promise<number | null>`
- `readStorage(storageType: string): Array<{ key: string; value: string }>`
- `writeStorage(storageType: string, key: string, value: string): void`
- `removeFromStorage(storageType: string, key: string): void`
- `importToStorage(storageType: string, entries: Record<string, string>, clearFirst: boolean): void`

Note: `writeStorage`, `removeFromStorage`, `importToStorage` are passed to `chrome.scripting.executeScript` as serialized functions — they cannot import modules. They must remain self-contained. They move to `src/lib/storage.ts` for organization but are still standalone functions.

`getActiveTabId` can import `chrome.tabs` normally since it runs in the extension context.
